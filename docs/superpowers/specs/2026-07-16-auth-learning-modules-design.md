# 巡防百科简易登录与学习模块扩展设计

日期：2026-07-16  
状态：待书面规格复核

## 1. 目标与范围

本次更新在现有纯静态“巡防百科”中渐进式增加四项能力：

1. 面向普通用户的通用账号密码登录；
2. 独立的“考核标准”模块；
3. 独立的“纪律学习”模块；
4. 独立的“巡防入门指南”模块及首页五阶段学习路线。

继续使用现有 HTML、CSS 和原生 JavaScript 架构，不重建旧栏目，不引入后端、第三方运行时、远程字体、统计脚本或个人学习进度系统。本次完成本地实现与验证，不主动发布到公网。

## 2. 已确认的产品决策

- 登录采用纯前端简易验证，只作为普通用户的访问门槛，不作为保密或安全边界。
- 通用账号固定为 `xunfang`；用户指定的通用密码只在实施时转为校验摘要，不在新增源码或文档中保存明文。
- 登录成功后返回原访问页面，登录状态保留 24 小时。
- 三个新模块分别显示为首页卡片、导航入口和独立栏目，不集中到“学习中心”或下拉菜单。
- “巡防入门指南”首页采用五阶段路线，不记录个人进度。
- “考核标准”和“纪律学习”各提供 4 至 6 篇首期文章。
- 内容只使用当前有效、可以公开核实的权威来源；无法公开核实的内部指标不写入正文。
- 登录背景采用原创的电影感夜间特警巡防画面。

## 3. 信息架构

新增三个目录：

- `rumen/`：巡防入门指南、阶段目录和编号课程；
- `kaohe/`：考核标准栏目页与文章；
- `jilv/`：纪律学习栏目页与文章。

首页结构调整为：

1. 原有顶部导航；
2. 搜索入口；
3. 五阶段巡防入门路线；
4. 包含三个新模块在内的独立模块卡片网格；
5. 原有每月一学等内容。

桌面端导航改为品牌栏和可换行的模块导航，三个新模块各自拥有入口。手机端继续使用折叠菜单，每个模块独占一行。不得把三个模块放入同一个“学习中心”入口或下拉分组。

## 4. 简易登录设计

### 4.1 登录流程

所有受保护页面在显示正文前加载共享登录守卫。登录页是唯一不执行守卫的页面。

1. 守卫检查全站登录 Cookie 及其有效状态；
2. 未登录用户跳转到 `auth.html?redirect=<站内地址>`；
3. 用户输入固定通用账号和密码；
4. 浏览器计算校验摘要并与代码中的摘要比较；
5. 验证成功后写入有效期为 86,400 秒的全站 Cookie；
6. 用户返回原访问页面；
7. 主动退出或 Cookie 过期后重新进入登录页。

跳转参数只能是当前站点内的相对路径或同源地址。协议相对地址、外部域名、脚本协议和无法解析的地址一律回退到首页。HTTPS 部署时 Cookie 使用 `Secure`，并始终使用 `Path=/` 和 `SameSite=Lax`。

### 4.2 登录页交互

- 账号框、密码框、显示或隐藏密码按钮、登录按钮和错误提示；
- 支持回车提交；
- 空账号、空密码、错误账号或密码均不登录；
- 失败提示不区分账号错误和密码错误；
- 校验期间避免重复提交；
- 不支持必要浏览器加密能力时显示兼容性说明；
- 导航提供“退出登录”，退出后删除 Cookie 并返回登录页。

### 4.3 安全边界

前端代码、摘要和 Cookie 均可被技术用户检查或绕过。登录页及站内说明必须明确：此功能不能保护涉密、工作秘密或真正敏感内容。所有新增内容即使处于登录后，也必须满足公开传播标准。

## 5. 登录页视觉设计

生成一张原创的 16:9 电影感夜间巡防背景：城市街道、巡防车辆和特警巡逻剪影，采用深警蓝、冷白和少量暖橙灯光。主体位于画面右侧，桌面端登录卡片位于左侧；手机端卡片居中并重新裁切背景。

背景不得出现可识别面孔、真实单位徽章、文字、车牌、敏感武器细节或具体战术队形。画面添加深色渐变遮罩，保证表单和错误提示具有足够对比度。登录卡片采用克制的磨砂效果，并提供实色后备背景。

交付压缩后的 WebP 与 JPEG 后备图片。CSS 同时提供纯色警蓝渐变，确保图片加载失败时仍可登录。不使用持续闪烁的警灯动画，并尊重 `prefers-reduced-motion`。

## 6. 巡防入门指南

结构参考 [52codex.site 的“上手指南”](https://www.52codex.site/#hot-posts) 所采用的编号路线、明确学习顺序和简短课程说明，但不复制其文字、品牌、样式或页面代码。

五个学习阶段为：

1. 认识巡防岗位与职责边界；
2. 纪律要求与保密意识；
3. 勤务基础与群众沟通；
4. 训练要求与考核准备；
5. 综合复习与入门自测。

指南首页显示阶段顺序、阶段目标、预计阅读时间和“开始学习”。课程页显示编号、所属阶段、面包屑、上一课、下一课和返回路线。首期不保存已学状态、完成比例、成绩或个人数据。

## 7. 考核标准模块

首期提供 4 至 6 篇文章，选题范围为：

- 考核标准模块说明；
- 公开训练规范导读；
- 理论学习考核认知；
- 基础体能考核认知；
- 警务技能考核认知；
- 日常训练与考核准备。

文章可以介绍公开发布的国家、行业或地方训练规范。只有市局或支队已经公开发布并可从权威站点核实的内容，才能按其公开表述引用。无法公开核实的内部项目、分值、阈值、流程和材料不得编写或推测，页面统一提示“具体要求以本单位最新正式规定和组织安排为准”。

## 8. 纪律学习模块

首期提供 4 至 6 篇文章，选题范围为：

- 纪律学习模块说明；
- 人民警察职业纪律导读；
- 内务与日常行为规范；
- 保密与网络使用纪律；
- 执勤履职纪律；
- 典型认识误区与自测。

正文以实施时核验为现行有效的公开法律法规、部门规章和官方解读为依据。不得引用来源不明的内部课件、群聊文件、培训截图或无法确认公开属性的材料。

## 9. 统一文章结构与来源规则

新模块文章统一包含：

1. 学习目标；
2. 三至五条核心要点；
3. 与入门工作的关系；
4. 公开可以讲解的原则；
5. 常见认识误区；
6. 必须请示或接受正式培训的边界；
7. 非战术性自测题与解析；
8. 来源卡片；
9. 内容更新时间。

来源卡片记录发布单位、文件或页面名称、发布日期、本站核验日期和 HTTPS 原始链接。外部链接在新窗口打开并使用 `rel="noopener noreferrer"`。来源失效、法规修订或关键结论无法继续核实时，页面标记“待复核”，停止展示具体数值或结论，直至重新核验。

## 10. 搜索与页面交互

- 三个新模块的栏目页和文章页加入现有搜索索引；
- 搜索结果显示所属模块，不收录登录页；
- 首页搜索可以发现新模块与新文章；
- 自测只在当前页面显示选择结果、正确答案和解析，不上传或保存；
- 所有课程在禁用 JavaScript 时仍可阅读；
- 页面键盘焦点清晰，交互目标适合手机触控；
- 手机端五阶段路线改为纵向排列。

## 11. 异常处理

- 登录字段为空或校验失败：保留账号输入，清空密码并显示统一错误；
- Cookie 过期或被删除：返回登录页并保留当前站内地址；
- 跳转地址无效：返回首页；
- 背景图片加载失败：显示警蓝渐变和实色登录卡片；
- 来源失效或内容有效性存疑：标为待复核，不展示无法确认的具体标准；
- 搜索索引加载失败：显示可理解的失败提示，模块导航仍可使用；
- 内容或资源不存在：保留返回首页和返回所属模块的明确入口。

## 12. 测试与验收

### 12.1 自动检查

- 所有正式网站 HTML 页面均接入登录守卫，登录页除外；
- 正确账号和密码可登录，空输入与错误输入被拒绝；
- 登录 Cookie 的有效期为 86,400 秒；
- 外部或危险跳转参数被拒绝；
- 退出操作删除登录状态；
- 三个新模块在首页和导航中分别具有独立入口；
- 每个内容模块满足计划文章数量，链接、来源和更新时间完整；
- 搜索索引包含所有新文章且不包含登录页；
- 来源链接使用 HTTPS，并具有发布单位和文件名称；
- 登录背景具有 WebP、JPEG 和 CSS 后备方案。

### 12.2 人工验收

- 桌面、手机和窄屏导航可用且无入口重叠；
- 登录成功后准确返回原页面；
- 登录 24 小时后重新要求认证；
- 登录背景裁切合理，表单和提示文字清晰；
- 指南阶段目录、上一课和下一课链接正确；
- 首页卡片、栏目页、文章页、搜索、自测和主题切换正常；
- 键盘操作、焦点状态和降低动态效果设置正常；
- 浏览器控制台无新增错误。
