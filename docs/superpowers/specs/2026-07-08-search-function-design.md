# 搜索功能设计文档

**日期:** 2026-07-08
**状态:** 待实施

---

## 概述

首页搜索框目前仅有 UI，没有实际搜索逻辑。实现客户端 JSON 索引 + 纯 JS 中文关键词搜索，覆盖全部 55 篇文章。

## 方案选型

**方案 A：预建 JSON 索引 + 纯 JS 搜索（选定）**

- 构建 `search-index.json`，包含所有文章元数据
- 搜索 JS 加载 JSON 后在内存中匹配
- 零依赖，离线可用，55 篇文章量级完全胜任

不选 Lunr.js（中文分词需额外处理，杀鸡牛刀），不选运行时 fetch（55 个请求太慢）。

## 搜索索引

### 文件：`/search-index.json`

```json
[
  {
    "title": "伸缩警棍",
    "module": "装备介绍",
    "path": "zhuangbei/shensuo-jinggun.html",
    "desc": "伸缩警棍的击打部位、戒备姿势与操作流程",
    "tags": ["驱逐性警械", "伸缩警棍", "警械使用"],
    "keywords": ["开棍", "劈击", "戒备", "击打部位", "警棍"]
  }
]
```

字段说明：

| 字段 | 用途 |
|------|------|
| `title` | 文章标题（无 emoji），匹配权重 +5 |
| `module` | 所属模块名，结果显示用 |
| `path` | 相对根目录路径，跳转用 |
| `desc` | 一句话描述，结果摘要 + 匹配权重 +1 |
| `tags` | 页面标签数组，匹配权重 +3 |
| `keywords` | 补充的口语化/同义搜索词，匹配权重 +3 |

预估 55 条 ≈ 8-12 KB。

## 搜索 UI/UX

### 下拉面板（首页搜索框）

- **输入即搜**：200ms 防抖后实时展示结果
- **结果卡片**：标题 + 模块标签 + 描述摘要，最多 8 条
- **匹配高亮**：标题和描述中匹配到的关键词加粗标记
- **「查看更多」链接**：跳转到搜索结果全页
- **ESC 关闭**：按 ESC 或点击搜索框外部关闭下拉
- **无结果**：显示提示文字，建议换关键词

### 搜索结果全页

- **路径**：`/search.html?q=关键词`
- **展示**：全部匹配结果，按匹配分数排序
- **每项**：标题（可点击）+ 模块名 + 描述 + 匹配标签
- **返回**：面包屑链接回首页
- **无结果**：空状态提示

## 匹配算法

```
对每条索引记录计算匹配分数：
  title 包含关键词    → +5 分
  tags 包含关键词     → +3 分
  keywords 包含关键词 → +3 分
  desc 包含关键词     → +1 分
→ 过滤 0 分记录 → 按分数降序排列
```

- 大小写不敏感
- 子串匹配（输入"警棍"可匹配"伸缩警棍"）
- 多关键词时每条关键词独立计分累加

## 路径处理

- `search.js` 自动检测当前页面深度
- 首页调用时，结果路径直接使用（如 `zhuangbei/shensuo-jinggun.html`）
- 子页面调用时，自动加 `../` 前缀
- `search.html` 调用时，结果路径直接使用

## 错误处理

- `search-index.json` 加载失败：搜索框保持可用但提示"搜索暂不可用"，不阻塞页面
- 空输入：不触发搜索，不显示下拉面板
- JSON 解析失败：同加载失败处理
