# “巡防入门学习手册”双轨建设设计

日期：2026-07-14
状态：待用户复核

## 1. 背景与目标

现有“巡防百科”包含警情处置、巡防勤务、警务训练、装备、法规和走访送教等内容，其中部分主题可能涉及国家秘密、工作秘密、内部流程或不适合在互联网汇编传播的信息。现有网站是纯静态站，浏览器端密码验证不能构成可靠的保密边界；删除页面也不能自动清除 Git 历史、旧部署、搜索索引和媒体文件中的内容。

项目采用双轨路线：

1. 冻结现有完整网站，将其作为未来公安内网平台的候选母版，实施受控封存；
2. 在 `F:\frank第二大脑\xunfang-learning-handbook` 新建独立公开版项目“巡防入门学习手册”；
3. 公开版不继承现有仓库的 Git 历史、旧文章、旧附件或旧搜索索引；
4. 公开版先作为离线演示项目，经过业务、保密、宣传与版权审核以及技术检查后，才恢复公网部署。

## 2. 产品定位

产品名称：**巡防入门学习手册**

建议副标题：**面向新警、辅警的公开型基础知识与职业素养学习平台**

核心受众是新警、辅警和准备接触基层巡防工作的入门人员，社会公众是次要读者。平台帮助学习者建立岗位认知、纪律意识、法律意识、群众工作意识和风险意识，但不替代单位正式培训、内部规定和实战训练，也不提供可照着执行的战术教程。

## 3. 总体架构

### 3.1 轨道一：现有完整版本

现有 `xunfang-baike` 停止公网部署和新增业务内容，形成受控封存包。封存包是未来内网迁移的候选输入，不代表其中所有内容都已获准进入公安内网。

封存包应包含：

- 网站源文件与 Git 版本历史；
- 图片、音频、视频、PPT 等媒体和附件；
- 微信小程序及其数据；
- 搜索索引；
- 部署配置说明；
- 文件清单、校验值、封存日期、版本号、保管人和存储位置。

封存包只能进入单位认可的介质或环境，不上传到个人网盘、普通云盘或新的公共代码仓库。迁入内网前仍需确认系统密级、知悉范围、下载权限、日志策略和可存储内容范围。

### 3.2 轨道二：公开版新项目

公开版位于：

`F:\frank第二大脑\xunfang-learning-handbook`

该目录建立全新的独立 Git 仓库，不复制现有 `.git` 目录，不批量迁移旧文章、附件和搜索索引。通用页面结构、样式或代码只有在检查后才能重新采用。公开站不设置隐藏的内部专区，也不使用前端密码保存非公开内容。

### 3.3 内容审查台账

审查台账独立于公开网站，记录文章、图片、视频、音频、PPT、PDF、搜索摘要、小程序数据、HTML 注释和历史版本的处理结果。

每项内容采用四级标记：

- 绿色：公开来源明确，可以进入公开版；
- 黄色：主题可以保留，但必须完全重写并经过审核；
- 红色：涉及战术、部署、内部流程、装备细节或非公开案例，只进入内网候选版；
- 灰色：无法判断，停止使用并提交单位审查。

台账至少记录原内容、处理等级、公开依据、处理方式、来源、审核人、审核日期、审核意见和发布状态。台账本身不进入公开仓库。

## 4. 公开版信息架构

公开版设置九个栏目：

1. **初识巡防**：岗位价值、职责边界、工作场景、协作关系和常用术语；
2. **职业规范**：职业道德、纪律意识、保密意识、文明履职和形象规范；
3. **法律基础**：常用公开法律概念、权限边界、程序意识和权利保障；
4. **群众工作**：倾听、解释、安抚、矛盾调解意识、特殊群体沟通和服务礼仪；
5. **勤务常识**：上岗准备、协作、请示、记录和交接意识，只讲原则，不公开部署和内部流程；
6. **风险识别**：环境观察、常见风险信号、自我保护意识和及时报告；
7. **装备认知**：装备分类、保管责任、安全责任和依法规范使用原则，不提供敏感参数、弱点或操作教程；
8. **情景课堂**：使用虚构、公开或充分泛化的场景讲解法律、沟通、纪律和风险原则；
9. **学习中心**：每月一学、资料导航、术语解释、知识检索、视频导读和自测。

首页提供五阶段学习路线：认识岗位、纪律与法律、沟通与勤务、风险与装备、情景练习与自测。用户也可以按栏目学习、搜索公开知识或进入“每月一学”。

## 5. 内容产品与首期范围

每个栏目持续生产四类内容：

- 基础课：约五分钟阅读量的入门主题；
- 一图看懂：适合手机查看的原创知识图卡；
- 情景练习：围绕原则、请示和报告边界设计的非战术性练习；
- 权威导读：介绍官方文章、法规或公开视频并链接原始来源。

每篇基础课采用统一结构：

1. 三至五条核心结论；
2. 与入门工作的关系；
3. 可以公开讲解的原则；
4. 虚构、公开或充分泛化的情景；
5. 常见认识误区；
6. 必须请示或接受正式培训的边界；
7. 原创图解或知识卡；
8. 官方视频导读；
9. 三至五道非战术性自测题；
10. 来源卡片。

首期目标约 30 项内容：9 篇栏目导读、8 篇基础课、6 张原创图解、3 个视频导读、3 组情景练习和 1 期“每月一学”。第一期不要求九个栏目全部达到相同深度，但每个栏目至少有清晰入口和定位说明。

## 6. 互联网素材供应链

互联网用于发现公开素材，不能成为自动转载来源。候选素材按照以下流程处理：

权威公开来源 → 候选素材池 → 实用性筛选 → 保密与聚合风险检查 → 版权与转载条件检查 → 重新编写和原创图解 → 人工复核 → 定期发布 → 更新、纠错或下架。

### 6.1 来源等级

一级来源可作为事实依据，包括国家法律法规数据库、中国人大网、中国政府网，公安部、司法部、应急管理部等官方网站，各级公安机关正式门户和认证账号，人民法院、人民检察院等权威平台，以及正式发布的公开标准和教材。

二级来源只能作为选题线索，包括权威新闻媒体、政法院校或警察院校公开课程、专业期刊公开摘要，以及有明确作者和出处的专业科普内容。重要结论应尽可能回到法律原文、正式文件或官方发布核验。

无法确认作者和出处的自媒体文章、网盘课件、群聊文件、内部培训截图、标注“内部资料”或“仅供交流”的内容，以及未确认发布权限的案件录像和执法视频，不直接采用。

### 6.2 发布方式

- 原创讲解型：根据多个权威公开来源重新组织内容，是主要方式；
- 适当引用型：只引用解释问题所必需的少量内容，并注明作者或发布单位、作品名称和链接；
- 链接导读型：本站提供学习重点，引导用户前往原平台阅读或观看；
- 授权转载型：取得明确许可或来源明确允许转载后，保存授权记录再发布。

官方视频优先使用原平台嵌入或跳转链接；无法确认许可时不下载重传。图解优先自行绘制。图片和视频发布前检查人脸、车牌、证件、定位、单位布局和文件元数据。

即使单条信息已经公开，汇编后若可能暴露部署规律、应对弱点、处置能力或内部判断逻辑，也不得进入公开版。自动化只能用于发现资料、抓取标题、记录链接和提醒更新，不能自动发布。

建议每周筛选 5 至 10 条候选资料，每两周发布 1 篇基础课和 1 张知识图卡，每月发布 1 期“每月一学”；法规发生变化时触发专项复核，每季度检查失效链接和过期内容。

## 7. 禁止公开的内容

公开版不得包含：

- 具体战术动作、站位、控制步骤和处置预案；
- 巡逻部署、勤务规律、力量配置和响应机制；
- 装备敏感参数、性能弱点、详细操作方法或使用条件；
- 内部文件、未公开术语、未公开培训材料；
- 真实案件中的身份信息、地点规律和内部复盘；
- 系统地址、账号规则、权限结构或其他安全措施；
- 仅以“内部交流”“请勿外传”为来源的内容；
- 公开碎片汇编后能够形成的新敏感信息。

## 8. 审核与发布流程

所有内容执行“三审一检”：

1. 业务审核：判断准确性、实用性以及是否会误导学习者；
2. 保密审核：判断国家秘密、工作秘密、内部流程和聚合风险；
3. 宣传与版权审核：判断公开表达、主体身份、转载许可、图片视频和来源标注；
4. 技术检查：检查旧文件、隐藏内容、元数据、搜索索引、Git 历史、直接 URL 和部署文件。

稿件状态依次为：候选素材、编辑中、待业务审核、待保密审核、待宣传版权审核、待发布、已发布、已修订或已下架。只有“已发布”稿件才能进入网站和公开搜索索引。

审核原则上由不同人员承担，撰稿人不得独自完成全部审核。业务、保密、宣传与版权审核以及技术检查全部通过，并经领导确认后，才允许恢复公网部署。

## 9. 分阶段实施流程

### 第 0 阶段：停止公网暴露

暂停现有部署，检查 GitHub 仓库可见性、部署平台、域名、历史预览地址和分享链接，验证首页、具体页面、媒体文件和搜索索引不能直接访问，并暂停自动部署。

### 第 1 阶段：受控封存

在不破坏现有母版的前提下生成文件清单和校验值，由单位确定存储介质、保管人和移交流程。

### 第 2 阶段：内容分级

逐页、逐附件建立审查台账，将内容标记为绿、黄、红或灰，明确处理方式和公开依据。

### 第 3 阶段：公开版重建

新建 `xunfang-learning-handbook` 独立目录和独立 Git 仓库，从经过审核的稿件与资产构建公开版，不从旧站批量迁移内容。

### 第 4 阶段：首期内容生产

完成约 30 项首发内容以及首页学习路线、栏目入口、搜索、来源卡片和自测功能。

### 第 5 阶段：审核与上线

完成三审一检，验证旧站和旧资源已经停止公开访问，经领导确认后再恢复公网部署。

### 第 6 阶段：持续运营

每月复核新增内容，每季度检查全站内容、来源和链接，每年至少进行一次完整审核。

## 10. 异常处理

- 无法判断内容性质时标为灰色，停止使用；
- 来源失效或关键结论无法核实时，暂停相关展示；
- 法规修订后，旧文章显示待复核状态并停止作为有效指引；
- 收到版权异议时，先下架争议素材再核查授权；
- 发现误发布时，立即停止公开访问、保留必要记录，并按单位制度报告；
- 审核意见冲突时采用更严格意见并提交领导决定；
- 图片或视频无法确认授权时，只保留原创文字导读和原始链接。

## 11. 发布前验证

恢复公网部署前必须确认：

- 旧站具体页面、附件和搜索索引均不能公开访问；
- 新仓库不包含旧 Git 历史和未审核资产；
- 不存在红色或灰色内容；
- 每项公开内容都有来源和完整审核记录；
- 图片视频不存在敏感画面和多余元数据；
- 搜索索引只收录审核通过的内容；
- 手机端、桌面端、导航、搜索、视频链接和自测功能正常；
- 网站显著说明平台定位、公开边界和不能替代正式培训；
- 业务、保密、宣传与版权审核、技术检查及领导确认全部完成。

## 12. 成功标准

项目完成应同时满足：

1. 当前完整版本已经停止公网暴露并形成受控封存方案；
2. 公开版使用独立目录、独立仓库和干净历史；
3. 首期约 30 项内容具有明确来源、实用价值和图文视频组合；
4. 不包含战术、部署、内部流程、敏感装备信息或非公开案例；
5. 三审一检可追溯，未经审核的内容无法进入公开搜索和部署；
6. 网站在桌面与窄屏设备上可用，来源链接、媒体和自测功能正常；
7. 只有在全部审核通过后才恢复公网部署。
