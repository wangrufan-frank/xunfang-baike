# 巡防百科公开资料索引设计

日期：2026-07-14
状态：待用户复核

## 1. 背景与目标

“巡防百科”现有静态网站包含 27 个独立内容页。项目需要为页面中的实质性知识点建立公开互联网资料对照，使读者和审核人员能够看到互联网上存在的相似公开信息，并在仓库内形成可追溯的来源证据台账。

本功能采用“知识点旁编号引用 + 页面底部完整来源列表”的呈现方式，并以结构化台账作为唯一维护入口，由生成工具把引用和来源列表写入静态 HTML。

公开资料索引只能证明检索时互联网上存在相似公开资料，不能证明本站文字实际转载自该资料，也不能自动证明本站内容不涉密、适合公开或已经通过主管部门审核。根据现行《中华人民共和国保守国家秘密法实施条例》第四十二条关于“先审查、后公开”和逐项审查的要求，来源核验与保密审查必须保持为两个独立状态。

## 2. 范围

本期只覆盖 `xunfang-baike` 静态网站，不修改微信小程序。

纳入范围的是以下 27 个非 `index.html` 内容页：

- `fagui/`：6 页；
- `xunlian/`：11 页；
- `zhuangbei/`：7 页；
- `zoufang/`：3 页。

`jingqing/`、`qinwu/` 和 `meiyueyixue/` 当前只有“内容整改中”列表页，不添加空来源区。首页、搜索页、认证页和各模块列表页不作为知识内容页处理。

本期不下载、镜像或重新发布外部网页全文、图片和视频，也不把外部来源内容复制进本站。

## 3. 方案选择

采用“统一证据台账 + 自动生成静态引用”方案。

不采用逐页手工维护，因为页面引用编号、页面底部列表和独立台账容易失去同步。不采用浏览器运行时加载，因为 JavaScript 或数据请求失败会使来源信息消失，不适合作为稳定的审核证据。

生成后的引用和来源列表直接存在于 HTML 中，不依赖 JavaScript。结构化台账是来源数据和知识点映射的唯一事实来源；生成工具可以重复运行，并且不会产生重复标记。

## 4. 来源选择与核验规则

### 4.1 来源优先级

来源按以下顺序选择：

1. 国家法律法规数据库、中国政府网、中国人大网以及政府部门正式网站；
2. 公安、司法、应急、教育等机关的门户网站或可确认主体的官方账号；
3. 正式公开标准、权威教材、院校或专业机构公开资料；
4. 微信公众号、百度百科、知乎等用于补充相似公开表达。

涉及法律结论、执法程序、警械使用条件或其他高风险陈述时，优先关联第一或第二级来源；微信公众号、百度百科、知乎等可以作为相似知识表达的补充。来源索引只证明存在相似公开内容，不承担逐句证明本站结论正确的功能。

### 4.2 正式来源的准入条件

一个来源只有同时满足以下条件，才可在页面显示为“已核验”：

- 审核人员实际打开并阅读了正文，而不只是看到搜索结果摘要；
- 页面标题、发布主体或作者、URL 可以确认；
- 正文与被引用知识点存在可说明的实质性相似内容；
- 记录核验日期；
- 页面未明确标记为内部资料、仅供交流、禁止外传或其他不适合公开传播的材料。

需要登录、正文无法打开、只有搜索摘要或来源身份无法确认的候选项，可以保留在内部候选记录中，但不得生成到页面正式来源列表。

### 4.3 覆盖标准

每个 `.step-card` 视为一个一级知识点。每个一级知识点至少关联一个实际打开正文并确认存在主题相似、概念相似或局部内容相似的公开来源；一个来源可以关联同页多个知识点，一个知识点也可以关联多个来源。

`coverage_status: "verified"` 表示“知识点与所列公开网页的相似关系已经人工核验”，不表示来源完整支持整段文字，也不表示页面结论正确。来源只覆盖部分陈述或与页面存在差异时，仍可标记为 `verified`，但“相似内容说明”必须明确写出相似点和必要差异，不得暗示整段内容得到权威背书。只有完全找不到可说明的相似公开内容时才保持 `pending`。

## 5. 页面设计

### 5.1 知识点旁引用

每个 `.step-card` 的 `.step-lead` 末尾显示引用标记，例如 `[1] [2]`。编号以页面为单位，从 1 连续排列，并与页面底部来源列表一一对应。

引用标记使用可点击锚点，跳转到本页对应来源条目。来源条目提供返回引用位置的链接。键盘用户可以聚焦引用链接，窄屏下引用不会溢出卡片。

如果同一来源支持多个知识点，各知识点重复显示同一个编号，不为同一 URL 创建重复来源条目。

### 5.2 页面底部来源区

来源区位于最后一个知识卡片之后、上一篇/下一篇导航之前，标题为“公开资料对照”。每条来源显示：

- 页面内编号；
- 来源标题；
- 发布主体或作者；
- 平台或网站名称；
- 发布日期（能够可靠取得时）；
- 核验日期；
- 原文链接；
- 相似内容说明。

来源链接在新窗口打开，并使用安全的 `rel` 属性。失效或待复核来源不得继续作为“已核验”来源显示。

### 5.3 固定说明

每个来源区显示以下统一说明：

> 本索引用于说明核验日期时互联网上存在与本页知识点相似的公开资料，不表示本站内容均转载自所列网页，也不构成保密审查、法律审查或业务主管部门审核结论。公开发布仍须执行本单位规定的先审查、后公开程序。

页面不得使用“内容来源于互联网，因此不涉密”或“已规避涉密责任”等绝对化表述。

## 6. 证据台账

仓库维护一个机器可读的来源台账和一个便于人工复核的汇总报告。机器可读台账至少包含以下实体：

### 6.1 页面记录

- `page_id`：稳定页面标识；
- `path`：相对 HTML 路径；
- `title`：页面标题；
- `review_status`：保密审查状态，值为 `pending`、`approved` 或 `rejected`；
- `reviewed_by`、`reviewed_at`：由有权人员填写，不得由来源生成工具推断。

### 6.2 知识点记录

- `point_id`：页面内稳定标识；
- `selector`：与页面知识卡片对应的稳定选择器；
- `label`：知识点标题；
- `source_ids`：对应来源标识列表；
- `coverage_status`：`verified` 表示相似关系已核验，`pending` 表示尚未找到或尚未打开核验相似网页；
- `coverage_note`：具体相似点以及必要的差异说明。

### 6.3 来源记录

- `source_id`：稳定来源标识；
- `title`：网页标题；
- `publisher`：发布主体或作者；
- `platform`：网站或平台；
- `url`：原文地址；
- `published_at`：发布日期，可为空；
- `verified_at`：实际打开正文的核验日期；
- `verification_status`：`verified`、`unavailable` 或 `needs_review`；
- `similarity_note`：简短的相似内容说明；
- `source_level`：来源等级；
- `last_checked_at`：最后一次可访问性检查日期。

台账不保存外部文章全文，不保存超出核验所需的大段摘录，也不把“公开来源已核验”自动映射为“保密审查已通过”。

## 7. 生成与数据流

数据流为：

1. 人工检索并打开候选网页；
2. 核对正文与具体知识点的相似关系；
3. 在证据台账中录入来源、映射、核验日期和覆盖说明；
4. 运行校验，阻止缺少来源、字段不完整或存在悬空映射的数据；
5. 运行生成工具，将知识点引用和来源区写入 27 个静态 HTML；
6. 生成供人工审核的全站来源汇总报告；
7. 人工完成业务、保密和发布审核；
8. 通过后再进入部署流程。

生成工具只管理带有明确开始和结束标记的来源片段。重复运行时替换受管片段，不修改知识正文、导航、媒体或其他现有结构。

### 7.1 后续内容更新门禁

来源索引是新增内容的强制组成部分，而不是上线后的补录事项。以后新增内容页或新增 `.step-card` 时，提交者必须在同一次变更中登记知识点、提供实际核验的来源路径、填写相似内容说明和核验日期，并重新生成静态引用。

校验工具必须对比当前 HTML 与台账：发现新页面、新知识点、标题变化、删除后遗留映射或未核验来源时直接失败，并报告具体页面与知识点。只有台账、来源覆盖和生成结果同步更新后，来源覆盖检查才能通过；保密审核仍使用独立发布门禁。

## 8. 审查状态与发布门槛

系统保留两个独立判断：

- `coverage_status`/`verification_status` 说明公开相似资料是否已经找到并核验；
- `review_status` 说明页面是否经过有权人员的保密审查。

生成工具无权将 `review_status` 改为 `approved`。即使 27 页全部具有公开相似来源，只要页面保密审查仍为 `pending` 或 `rejected`，就不能据此认定适合公网发布。

盾叉协同、控制动作、战术站位、武力升级、装备操作、具体处置步骤等聚合风险较高的内容，必须单独提交人工审查。多个公开碎片汇编后可能形成新的敏感信息时，应删除、泛化或转入受控环境，不能依靠增加更多来源链接解决。

## 9. 异常处理

- 来源打不开：标记为 `unavailable`，从正式生成结果中移除，并寻找替代来源；
- 来源内容变更：重新核验相似性并更新核验日期；
- 法律法规修订：相关来源及页面知识点全部进入 `needs_review`；
- 来源只覆盖部分内容或存在差异：保留相似来源，在说明中明确相似点与差异，不把索引表述成完整事实证明；
- 无法找到公开相似内容：保持 `pending`，提交内容删除、泛化或受控使用决定；
- 页面结构变化导致选择器失效：生成失败并报告具体页面和知识点，不静默跳过；
- 人工审查否决：保留必要的内部审查记录，但不得因为存在公开来源而覆盖否决结果。

## 10. 验证要求

自动检查至少验证：

- 27 个内容页全部存在台账页面记录；
- 每个 `.step-card` 都有稳定知识点映射；
- 每个实质性知识点至少关联一个 `verified` 来源；
- 所有引用编号均能跳转到存在的来源条目；
- 同页编号连续且同一来源不重复；
- 正式来源的标题、发布主体、URL、核验日期和相似内容说明不为空；
- HTML 中只存在一个受管来源区，重复生成不产生差异；
- 搜索索引生成流程不会把固定免责声明当作正文关键词重复收录。

人工检查至少覆盖：

- 桌面与窄屏下引用和来源区可读；
- 键盘跳转和返回引用可用；
- 外部链接打开正确；
- 相似内容说明与原网页正文相符；
- 高风险页面已完成业务和保密审查；
- 页面没有出现“有公开来源即不涉密”的误导性表达。

## 11. 成功标准

项目完成应同时满足：

1. 27 个内容页的每个实质性知识点均有可核验的编号引用；
2. 每页底部存在完整、静态、可访问的“公开资料对照”；
3. 证据台账能够从页面追溯到知识点、来源和核验记录；
4. 生成过程可重复，且能够自动发现缺失、重复和悬空引用；
5. 官方来源优先，高风险结论不依赖单一自媒体来源；
6. 来源核验与保密审查在数据和流程上明确分离；
7. 未经有权人员完成保密审查的页面，不因存在互联网相似内容而被自动认定为可公开；
8. 后续新增页面或知识点未同步提供已核验来源路径时，自动检查明确失败并阻止形成合格版本。
